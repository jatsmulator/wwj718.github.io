Open edX docker生产环境

Date 五 10 八月 2018 By wuwenjie Category edx. Tags edx

我相信open edx的容器化将对它的生态扩张，起到极大的推动作用，这种推动作用连同前后端分离，将成为open edx近来最重要的两大进展。关于open edx的这些重大调整，我们将在8月18日(下周末)北京的线下活动:Beijing Open edX Meetup 2018#1 中讨论

open edx容器化相关的谈论，我写在Open edX的容器化之路里

本文中我们将使用docker来安装Open edX的最新稳定版本:hawthorn.1, 用作生产环境

本文可以视使用regisb的openedx-docker项目的实践笔记

实验环境

  • 阿里云 香港节点
  • 2核/16G
  • ubuntu16.04

依赖

  • 安装docker
    • sudo gpasswd -a ${USER} docker
  • 安装docker-compose:
export LC_ALL=C
sudo pip install docker-compose

开始安装

git clone https://github.com/regisb/openedx-docker
cd openedx-docker/
make all

一个小时左右，一切就绪! all in one!

深入

openedx-docker同时支持生产模式和开发模式。

如果你熟悉open edx和docker，稍微读一下openedx-docker的readme，就可以开始正常的开发了

作者非常老司机，项目干净利落, 可读性极好，清澈见底，没隐藏任何魔法的东西。所有的东西基本走在：Makefile和docker-compose.yml写清楚了
