HMM(隐含马尔可夫模型)学习笔记

Date 六 20 八月 2016 By wuwenjie Category 数据. Tags data

隐含马尔可夫模型一直被认为是解决大多数自然语言处理问题最为快速有效的方法。它成功地解决了复杂的语音识别、机器翻译等问题。当我们看完这些复杂的问题是如何通过简单的模型描述和解决时，会不得不由衷地感叹数学模型之妙 --吴军《数学之美》

引

这部分引《数学之美》第五章的内容：

吴军博士在文章中写到：

自然语言是人类交流信息的工具，语言和通信的联系是天然的通信的本质就是一个编解码和传输的过程。但是自然语言处理早期的努力都集中在语法语义和知识表述上，离通信的原理越走越远，而这样离答案也就越来越远。当自然语言处理的问题回归到通信系统中的解码问题时，很多难题都迎刃而解了

通信模型

假设s1,s2,s3,...是信号源发出的信号，o1,o2,o3,...是接收器收到的信号。

那么如何根据观测端的接收信号来推测信号源的发出信号呢?

只需要从所有的源信息中找到最可能产生出观测信号的那个信息，用概率语言描述为：

在已知o1,o2,o3，...的情况下，求得令条件概率：

P(s1,s2,s3,...|o1,o2,o3,...)达到最大值的那个信息串s1,s2,s3,...即：

s1,s2,s3,... = Arg Max P(s1,s2,s3,...|o1,o2,o3,...)

其中Arg(Argument)表示P获得最大值对应的信息串。利用贝叶斯公式可以把上述公式变化为：

$$\frac{P(s1,s2,s3,...|o1,o2,o3,...) P(s1,s2,s3,...)}{P(o1,o2,o3,...)}$$

各项含义为:

  • P(s1,s2,s3,...|o1,o2,o3,...)：信息s1,s2,s3,...在传输后变为接收信号o1,o2,o3,...的可能性(这是一个条件概率，至于如何用自然语言描述，和问题域有关，P(A|B)中A，B不需要有因果关系)
  • P(s1,s2,s3,...): 表示s1,s2,s3,...本身是一个在接收端合乎情理的信号的可能性
  • P(o1,o2,o3,...): 表示在发送端产生信息o1,o2,o3,...的可能性(常数)

正本溯源

学习算法，去了解它的诞生和发展过程是个很好的学习方法.

到了19世纪，概率论的发展从对随机变量的研究发展到对随机变量的时间序列s1,s2,s3,...,st,..,即随机过程的研究，在任意时刻t，对于的状态st都是随机的。

马尔可夫提出了一种简单假设

$$P(st|s1,s3,s3,...,st-1)=P(st|st-1)$$

比如，对于天气预报，硬性假定气温只与昨天有关而与前台无关,这便是马尔可夫假设

推理

对于一个序列

$$X\{x_1, x_2, \ldots, x_N\}$$

整个序列的联合分布定义为：

$$P(X)=\prod_{i=1}^N{P(x_i|x_1,\ldots,x_{i-1})}$$

\(\prod\)是累乘
